LOYA Future Ideas and Features
This list is based on the current LOYA 4 codebase: Notes, Commands, Targets, Search, LOYA Terminal, Settings, Update, Recycle Bin, Diagnostics, and Recovery.
==================================================
1. High Priority Next Features
1. Workspaces / Profiles
- Add separate workspaces so one app can hold multiple independent environments.
- Each workspace should have its own database, settings, targets, history, backups, and update state.
- This would solve the future problem of mixing personal notes, work notes, labs, and customer projects in one place.
2. Pinned Dashboard / Home Page
- Add a real home page that shows pinned notes, recent commands, recent searches, last backup status, recycle-bin count, and update status.
- This should become the main quick-access screen when the app starts.
3. Multi-Select and Bulk Actions
- Add multi-select for notes, commands, and recycle-bin items.
- Bulk actions should include move group, delete, restore, export, tag add/remove, and duplicate.
- This is one of the biggest usability gaps in the current UI.
4. Better Note Editor
- Add autosave indicator, word count, line count, heading shortcuts, and note template support.
- Add note duplicate, note archive, and favorite note features.
- Add split view for edit and preview.
5. Attachments and Files
- Add support for attaching images, files, and screenshots to notes.
- Store attachment metadata in the database and files in a managed local folder.
- Add drag and drop support into the note editor.
==================================================
2. Notes and Groups
6. Drag and Drop Group Management
- Allow dragging notes between groups in Navigate.
- Allow dragging to Ungrouped.
- Allow drag reorder inside a group if manual order is added later.
7. Group Metadata
- Add group color, description, icon, and collapse defaults.
- Add group-level export and group-level search.
8. Note Templates
- Add built-in templates for checklists, command notes, meeting notes, incident notes, and research notes.
- Allow users to save any note as a reusable template.
9. Note History Viewer
- Expose a UI for note history from NotesHistory.
- Allow compare, preview old version, and restore selected revision.
10. Internal Links Upgrade
- Add back-links and linked from view.
- Add broken-link detection and automatic repair when notes are renamed or imported.
==================================================
3. Commands and Automation
11. Real Command Runner
- Add optional safe command execution from inside LOYA.
- Show command output, exit code, and execution history.
- Support copy, rerun, and save output to note.
12. Command Variables and Presets
- Extend targets with command presets and per-command parameter prompts.
- Let users fill variables through a small popup before running or copying a command.
13. Command Collections
- Add folders or categories for standalone commands.
- Allow grouping by platform, use case, or project.
14. Snippets with Parameters
- Upgrade snippets into reusable mini-templates with placeholders and quick fill forms.
==================================================
4. Search and Discovery
15. Full Text Search Upgrade
- Add fast full text indexing for notes and commands.
- Add highlight matches inside previews.
- Add search ranking by title, tags, group, and content relevance.
16. Smart Filters
- Add filters like favorite=yes, archived=no, has:attachment, updated:7d, created:30d, tag:any(...), group:any(...).
17. Saved Search Folders
- Allow organizing saved searches into folders.
- Add export/import for saved searches.
18. Global Command Palette
- Add one quick open box for notes, commands, targets, pages, and saved searches.
- Keyboard-first access would make LOYA much faster.
==================================================
5. Targets and Sensitive Data
19. Secret Targets
- Add hidden target values for passwords, API keys, tokens, and secrets.
- Hide them in UI by default and require reveal or copy action.
- Consider encrypting only sensitive values instead of the whole app state.
20. Target Validation
- Add value type validation such as IP, URL, domain, email, path, port, token, or custom regex.
- Show invalid target values before they break searches or commands.
21. Target Profiles
- Let one target hold multiple profiles like Dev, Test, Stage, and Prod.
- Switching profile should update values everywhere in LOYA.
==================================================
6. LOYA Terminal and Assistant
22. Natural Language Actions
- Let LOYA turn plain requests into structured searches, note creation, and command suggestions.
- Example: find my nginx notes from Ops and open the latest one.
23. Session Memory
- Add temporary session context so LOYA can keep a short chain of related requests.
- Example: after showing a note, now search commands for this target.
24. Action Confirmation Layer
- For any future destructive or execution action, add a consistent confirm/cancel flow in LOYA.
25. Better Help System
- Add interactive help, examples, and suggestion cards for each LOYA command.
==================================================
7. Data Safety and Maintenance
26. Scheduled Backups
- Add automatic daily or weekly backup jobs.
- Add backup cleanup policy and backup verification.
27. Restore Preview
- Before restoring a backup, show what will be replaced.
- Allow restoring only notes, only targets, or only settings.
28. Startup Repair Wizard
- Turn health-check warnings into a repair wizard with direct fix buttons.
- Example: recreate missing files, repair schema, reset bad settings section, or open recovery.
29. Export Audit
- Add export manifest with app version, export date, counts, and checksums.
==================================================
8. Sync and Sharing
30. Git Sync
- Allow syncing notes and exports to a git repo.
- Useful for technical users who already track notes in GitHub or private repos.
31. Cloud Sync
- Add optional sync providers later, but only after conflict handling is designed correctly.
32. Share Package
- Export a note or group with its linked commands and related targets in one package.
==================================================
9. UX and Interface Improvements
33. Better Navigation Layout
- Add resizeable sidebars, persistent pane sizes, and optional compact mode.
34. Tabs or Multi-Document View
- Allow opening multiple notes at the same time.
- Add note tabs or split panes.
35. Keyboard Shortcuts
- Add full keyboard support for page switching, open note, create note, save, search, and terminal focus.
36. Theme System
- Add light theme and custom theme presets.
- Move more hardcoded spacing and component styles into theme variables.
37. Notifications
- Add small non-blocking status toasts for save, restore, update, import, and recycle-bin actions.
==================================================
10. Update System Future Work
38. Release Channels
- Add Stable and Beta channels.
- Keep official-source validation, but let users choose the channel.
39. Delta Updates
- Reduce package size by updating only changed files when safe.
40. Update Changelog View
- Show release notes before install.
- Show changed version, backup name, and rollback path in the Update page.
41. Update Verification Hardening
- Add signed manifest or signed package verification on top of current repo and version checks.
==================================================
11. Packaging and Delivery
42. Real App Builds
- Add Windows packaged build, installer, or portable release.
- Later add Linux package or AppImage.
43. First Run Setup Wizard
- Guide users through security, backup, and initial target setup on first launch.
==================================================
12. Developer Quality
44. Automated Tests
- Add unit tests for database migration, import/export, recycle bin, updater validation, and note reference logic.
45. UI Smoke Tests
- Add offscreen Qt smoke tests for major pages and flows.
46. CI Pipeline
- Run lint, compile checks, tests, and release package validation automatically on GitHub.
47. Logging Cleanup
- Standardize logs by area: app, update, recovery, import, and backup.
48. Codebase Cleanup
- Continue consolidating repeated data access logic.
- Move more page-specific helper logic into shared modules where it reduces duplication.
==================================================
Recommended Next Version Focus
If LOYA 4 is already stable, the strongest LOYA 4.x roadmap is:
- Workspaces / Profiles
- Multi-select and bulk actions
- Real command runner
- Note templates and attachments
- Full text search upgrade
- Scheduled backups
- Keyboard shortcuts and command palette
- Automated tests and CI
